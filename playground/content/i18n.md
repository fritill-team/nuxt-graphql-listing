# Internationalization (i18n)

The module includes built-in support for internationalization with English and Arabic translations. It integrates seamlessly with `@nuxtjs/i18n`.

## Built-in Languages

The module ships with translations for:

- **English (en)** - Default
- **Arabic (ar)** - Right-to-left support included

## useListingI18n Composable

Access translations in your components:

```ts
import { useListingI18n } from '#imports'

const { t, locale } = useListingI18n()

// Use translations
const filtersLabel = t('listing.filters')  // "Filters"
const resultsText = t('listing.results', { from: 1, to: 20, total: 100 })
// "Showing 1 - 20 of 100"
```

## Customizing Translations

### Override in Nuxt i18n

Add or override translations in your app's i18n configuration:

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  i18n: {
    defaultLocale: 'en',
    locales: [
      { code: 'en', name: 'English' },
      { code: 'ar', name: 'Arabic', dir: 'rtl' },
      { code: 'fr', name: 'French' }
    ]
  }
})
```

### Add Translation Files

Create locale files in your project:

```ts
// i18n/locales/en.ts
export default {
  listing: {
    filters: 'Filters',
    apply: 'Apply',
    clear: 'Clear All',  // Override default
    categories: 'Categories',
    // ... add more overrides
  }
}
```


### Configure i18n Module

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  i18n: {
    defaultLocale: 'en',
    locales: [
      { code: 'en', file: 'en.ts' },
      { code: 'ar', file: 'ar.ts', dir: 'rtl' },
      { code: 'fr', file: 'fr.ts' }
    ],
    lazy: true,
    langDir: 'i18n/locales'
  }
})
```

## RTL Support

The module automatically handles RTL (right-to-left) layouts for Arabic and other RTL languages:

1. **Nuxt UI**: Components adapt to RTL automatically
2. **Tailwind CSS**: Use `rtl:` variants for custom styling
3. **Dir Attribute**: Set via i18n locale configuration

### RTL-Specific Styling

```vue
<template>
  <div class="flex rtl:flex-row-reverse">
    <ListingFiltersAsidePanel class="w-64 ltr:mr-4 rtl:ml-4" />
    <div class="flex-1">
      <!-- Content -->
    </div>
  </div>
</template>
```

## Using Translations in Custom Components

```vue
<template>
  <div>
    <h2>{{ t('listing.filters') }}</h2>
    <p>{{ t('listing.results', { from, to, total }) }}</p>
    <button @click="clearFilters">
      {{ t('listing.clear') }}
    </button>
  </div>
</template>

<script setup lang="ts">
const { t } = useListingI18n()

const from = computed(() => offset.value + 1)
const to = computed(() => Math.min(offset.value + limit.value, total.value))
</script>
```

## Dynamic Labels

For filter and sort labels, use the i18n `t()` function:

```ts
const filterConfig = computed(() => [
  {
    key: 'category',
    field: 'categoryId',
    kind: 'select',
    label: t('filters.category'),  // Translated label
    options: [
      { label: t('filters.allCategories'), value: null },
      { label: t('filters.electronics'), value: 'electronics' }
    ]
  }
])

const sortConfig = computed(() => [
  { label: t('sort.newest'), field: 'createdAt', direction: SortDirection.DESC },
  { label: t('sort.priceAsc'), field: 'price', direction: SortDirection.ASC },
  { label: t('sort.priceDesc'), field: 'price', direction: SortDirection.DESC }
])
```

## Locale Switching

Handle locale changes in your app:

```vue
<template>
  <USelect
    :model-value="locale"
    :options="availableLocales"
    @update:model-value="setLocale"
  />
</template>

<script setup lang="ts">
const { locale, locales, setLocale } = useI18n()

const availableLocales = computed(() =>
  locales.value.map(l => ({
    label: l.name,
    value: l.code
  }))
)
</script>
```

## Complete Example

```vue

      <UButtonGroup>

          :variant="locale === loc.code ? 'solid' : 'ghost'"
          @click="setLocale(loc.code)"
        >
          {{ loc.name }}
        </UButton>
      </UButtonGroup>
    </div>

    <!-- Listing with translated labels -->

      :items="items"
      :total="total"
      :loading="loading"
      :filters="filters"

      :sort="sort"
      :sort-config="sortConfig"

      @filter-change="setFilter"

          <UButton @click="resetFilters">
            {{ t('listing.empty.resetFilters') }}
          </UButton>
        </div>
      </template>
    </ListingLayout>
  </div>
</template>

<script setup lang="ts">
const { t, locale } = useListingI18n()

const { items, total, loading, filters, sort, setFilter, setSort } = useProductsListing()
```
